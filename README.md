# AI Skin Disease Detection Platform 
**with Blockchain-Secured Medical Image Storage**

A multi-user platform where users upload skin images, receive AI-based disease predictions (using MobileNetV2), and securely store images using AES-256 encryption, IPFS storage, and Ethereum blockchain integrity verification.

## 🚀 1. Setup & Installation

**Prerequisites:** Python 3.9+, Node.js (for Ganache), IPFS CLI

**Install Dependencies:**
```bash
# Optional: Create a virtual environment
python -m venv venv
# Windows: venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

# Install requirements
pip install -r requirements.txt
```

**Environment Variables:**
Replace the dummy configurations in the `.env` file at the root. Be sure to configure `AES_SECRET_KEY` exactly as a 32-byte string, and point `BLOCKCHAIN_RPC_URL` to your active local node or public testnet.

## 🧠 2. Training the AI Model (Manual Step)

The training script generates a customized MobileNetV2 model and outputs training metrics, plots, and a confusion matrix.

1. Download the **HAM10000 Dataset** and organize the images into folders by class name inside the `dataset/` directory (e.g., `dataset/melanoma/`, `dataset/nv/`).
2. Run the training script:
```bash
# Note: Ensure you are running this from inside the model/ directory or configure the paths appropriately
cd model
python train.py
```
This saves `skin_model.h5` into `trained_models/` and evaluation outputs into `model/eval_outputs/`.

## ⛓️ 3. Deploying the Smart Contract (Ganache)

We use Ganache for local blockchain testing.

1. Install Ganache globally if you don't have it: `npm install -g ganache`
2. Run Ganache:
```bash
ganache-cli -p 7545
```
3. Copy one of the private keys generated by Ganache and place it in the `.env` file (`PRIVATE_KEY=...`).
4. Deploy the contract:
```bash
python blockchain/deploy_contract.py
```
5. Copy the deployed **Contract Address** from the console output and update `CONTRACT_ADDRESS` in `.env`.

## 📦 4. Starting IPFS

1. Install IPFS Kubo.
2. Initialize IPFS: `ipfs init`
3. Run the IPFS Daemon:
```bash
ipfs daemon
```

*Note: The platform has fallback logic. If IPFS or Blockchain nodes are unreachable, it will store data locally and show appropriate warnings without breaking the app.*

## 💻 5. Running the Web Platform

Ensure you are in the project root directory. Start the Flask application:

```bash
python backend/app.py
```
Navigate to `http://127.0.0.1:5000` in your browser.

## 🔍 6. How Blockchain Verification Works

When a user uploads an image:
1. The raw image is hashed using **SHA-256**.
2. The image is encrypted using **AES-256** and uploaded to IPFS.
3. The original Hash, IPFS CID, User ID, and Image ID are anchored into an **Ethereum Smart Contract**.

During Verification:
1. The system downloads the encrypted file and decrypts it dynamically in-memory.
2. It hashes the decrypted output using **SHA-256**.
3. It fetches the original hash from the **Ethereum Blockchain**.
4. Both hashes are compared to cryptographically prove zero-tampering.
